前端面试高频知识点

高优先级

HTML/CSS

Q: 盒模型的区别及如何切换？

A: 标准盒模型（content-box）width/height 只包含内容区，边框和内边距不计入。IE 盒模型（border-box）width/height 包含内容、内边距和边框。切换方式：box-sizing: border-box;。

Q: Flex 布局常用属性及应用场景？

A: 常用属性有 display: flex、flex-direction、justify-content、align-items、flex。适合一维布局，如导航栏、卡片列表等。

Q: 实现一个三栏自适应布局（中间自适应，两侧定宽）？

A: 可用 Flex 布局：display: flex;，两侧定宽，flex: 1 让中间自适应。

JavaScript

Q: 事件委托原理及应用场景？

A: 事件委托利用事件冒泡，将子元素事件绑定到父元素上，减少事件监听数量，适合动态生成元素的场景。

Q: 深拷贝的实现方式？

A: 常用方法有 JSON.parse(JSON.stringify(obj))（不支持函数、循环引用），递归遍历属性，或使用结构化克隆算法（structuredClone）。

Q: this 的几种绑定方式？

A: 1. 默认绑定（全局/严格模式）；2. 隐式绑定（对象调用）；3. 显式绑定（call/apply/bind）；4. new 绑定（构造函数）。

Vue 技术栈

Q: Vue3 响应式原理是什么？在实际项目中如何优化性能？

A: Vue3 使用 Proxy 实现响应式，能更精细追踪依赖。实际项目中，避免不必要的响应式数据、合理拆分组件、使用 shallowReactive 优化性能。

Q: Vue 项目如何实现大型应用的模块化和解耦？

A: 采用 Vuex、Pinia 管理状态，使用动态路由和异步组件拆分业务模块。实际项目中可结合微前端架构提升可维护性。

Q: 组件通信方式有哪些？适用场景？

A: props/$emit（父子）、provide/inject（跨层级）、event bus/mitt（兄弟）、Vuex/Pinia（全局）。

Q: 生命周期钩子及常见用途？

A: created（数据初始化）、mounted（DOM 操作/第三方库）、beforeUnmount/unmounted（清理副作用）。

React

Q: React Hooks 的原理及在复杂业务中的最佳实践？

A: Hooks 通过闭包和依赖数组实现状态与副作用管理。复杂业务中自定义 Hook 复用逻辑，避免组件臃肿。

Q: React 性能优化常用手段？

A: 使用 React.memo、useMemo、useCallback 避免不必要渲染，代码分割（如 React.lazy）提升首屏速度。

Next.js

Q: Next.js 如何实现服务端渲染（SSR）？实际场景如何选择 SSR/SSG？

A: Next.js 通过 getServerSideProps 实现 SSR，getStaticProps 实现 SSG。电商首页适合 SSR，内容型页面适合 SSG。

TypeScript

Q: TypeScript 在大型项目中的优势及常见类型体操应用？

A: 提升代码可维护性和可读性，类型体操如条件类型、映射类型可实现复杂类型推导，提升开发效率。

AI 前端技术应用

Q: 如何在前端项目中集成 AI 能力（如 OpenAI API）？

A: 通过 RESTful API 调用 AI 服务，结合表单、聊天、智能推荐等场景。实际项目中需注意接口安全和数据脱敏。

中大厂高频算法题

1. 两数之和（LeetCode 1）

题目地址：https://leetcode.cn/problems/two-sum/
思路：用哈希表存储已遍历的数字，查找 target-当前值 是否存在。

2. LRU 缓存机制（LeetCode 146）

题目地址：https://leetcode.cn/problems/lru-cache/
思路：用哈希表+双向链表，O(1) 时间完成 get 和 put。

3. 无重复字符的最长子串（LeetCode 3）

题目地址：https://leetcode.cn/problems/longest-substring-without-repeating-characters/
思路：滑动窗口，维护窗口内无重复字符。

4. 二叉树的层序遍历（LeetCode 102）

题目地址：https://leetcode.cn/problems/binary-tree-level-order-traversal/
思路：BFS 队列实现。

5. 岛屿数量（LeetCode 200）

题目地址：https://leetcode.cn/problems/number-of-islands/
思路：DFS/BFS 遍历所有陆地。

中优先级

工程化

Q: 前端项目如何实现自动化部署与持续集成？

A: 使用 GitHub Actions、Jenkins 等工具自动化测试、构建、部署，提升团队协作效率。

Q: webpack 和 vite 的区别？

A: webpack 采用打包构建，vite 利用原生 ES 模块，开发环境冷启动快，生产环境也支持打包优化。

Q: 常见的 loader 和 plugin？

A: loader 如 babel-loader、css-loader，plugin 如 HtmlWebpackPlugin、DefinePlugin、MiniCssExtractPlugin。

Q: 如何实现按需加载？

A: 通过动态 import 语法（import()），结合路由懒加载、组件懒加载。

性能优化

Q: 如何分析和优化前端性能瓶颈？

A: 利用 Chrome DevTools、Lighthouse 分析性能，按需加载、资源压缩、懒加载优化体验。

Q: 前端性能优化常见手段？

A: 资源压缩与合并、懒加载、预加载、SSR/SSG、虚拟滚动、长列表优化、CDN 加速、缓存策略。

Q: 如何优化首屏加载速度？

A: 代码分割、图片压缩、骨架屏、服务端渲染、减少首屏资源体积。

网络

Q: HTTP1.1/2/3 区别？

A: HTTP2 多路复用、头部压缩、服务器推送；HTTP3 基于 QUIC 协议，连接更快。

Q: 常见状态码及含义？

A: 200（成功）、301/302（重定向）、401（未授权）、403（禁止）、404（未找到）、500（服务器错误）。

Q: 跨域的实现方式？

A: CORS、JSONP、postMessage、nginx 代理、iframe。

低优先级

安全

Q: 前端如何防范 CSRF 和 XSS？

A: CSRF 通过 Token 校验，XSS 通过输入转义和 CSP 策略防护。

Q: XSS 和 CSRF 的区别及防御？

A: XSS（跨站脚本）注入恶意脚本，防御：输入校验、输出转义、CSP。CSRF（跨站请求伪造）冒充用户发请求，防御：Token、SameSite Cookie、Referer 校验。

Q: 前端如何防止敏感信息泄露？

A: 不在前端存储敏感信息，接口加密、HTTPS、权限校验。

项目架构设计

Q: 五年经验如何设计高可维护性前端架构？

A: 采用分层架构、模块化、接口规范、单元测试，结合微前端提升可扩展性。

团队协作与技术选型

Q: 如何推动团队技术升级和选型？

A: 结合业务需求评估新技术，组织技术分享，推动代码规范和最佳实践落地。
